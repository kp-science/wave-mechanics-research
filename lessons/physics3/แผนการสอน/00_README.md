## 🗂 โครงสร้างของแต่ละแผน (10 หัวข้อ · ~10–15 หน้า/แผน)

1. **สาระสำคัญ** — ภาพรวมแนวคิดหลักของแผน
2. **มาตรฐาน · ตัวชี้วัด · จุดประสงค์การเรียนรู้ (KPA)** — แยก K · P · A พร้อมรหัสจุดประสงค์
3. **สมรรถนะสำคัญและคุณลักษณะอันพึงประสงค์** — ครบ 6 สมรรถนะ + 4 คุณลักษณะ
4. **แนวคิดคลาดเคลื่อน (Misconceptions) เป้าหมาย** — 4 ตัว/แผน พร้อมแนวคิดที่ถูกต้อง
5. **สาระการเรียนรู้** — เนื้อหาวิทยาศาสตร์ + สมการสำคัญ
6. **กระบวนการจัดการเรียนรู้ (5E + POE)** — 9–11 ขั้นย่อย พร้อม**บรรยาย วPA แต่ละขั้น**
   - TLC ก่อนเรียน → ACT (ทบทวน Kahoot) → GOAL (Visible Learning) → F-Pre (Four-tier)
   - E1 Engage (Concept Cartoon) → E2 Explore (Lab/Workshop) → E3 Explain (CER) → E4 Elaborate (Calc + Spot)
   - TLC หลังเรียน → F-Post → MJ (Metacognitive Journal 3-2-1)
7. **สื่อ อุปกรณ์ และแหล่งการเรียนรู้** — รวม Simulation URL + เครื่องมือทดลอง
8. **การวัดและประเมินผล** — ตารางครบ ด้าน·ตัวแปร·เครื่องมือ·เกณฑ์ + Lab/Workshop Rubric
9. **การเชื่อมโยงกับ วPA ด้านที่ 1 (8 ตัวชี้วัด)** — ตารางหลักฐานครบทุกตัวชี้วัด
10. **บันทึกหลังการสอน** — 6 หัวข้อย่อย พร้อมช่องว่างให้ครูเขียน + ลายเซ็น

## 🛠 หมายเหตุทางเทคนิค

- **Font:** TH Sarabun New 14 pt (default)
- **Alignment:** LEFT (ไม่ใช่ Justify) เพื่อป้องกันตัด Thai word breaks
- **Thai line-break fix:** ใช้ PyThaiNLP `word_tokenize(engine="newmm")` แทรก ZWSP (U+200B) ระหว่างคำ
- **Source:** สร้างจาก `*_Infographic.html` ในแต่ละ unit folder ด้วย `_generator.py`
- **ปรับแก้:** เปิด docx ใน Word แก้ไขได้โดยตรง · ถ้าจะ regenerate ใหม่ ให้แก้ infographic html แล้วรัน `python3 _generator.py`
- **DOCX ใน .gitignore:** ไฟล์ DOCX 40 ไฟล์ไม่ถูก push เข้า Git (เปลือง storage) · sync ผ่าน iCloud แทน
- **HTML ใน Git:** ไฟล์ Infographic HTML ทั้งหมด push เข้า Git → ครูเครื่องอื่น `git pull` แล้วรัน `_generator.py` เพื่อผลิต DOCX ใหม่ได้
