🎨 Pattern Library

แต่ละวิชามี pattern (โครงสร้าง + template + คำถามที่ต้องเตรียม) เวลาเปิดวิชาใหม่ให้เลือก pattern ที่อยากใช้ · ระบบถามข้อมูลชุดเดียวแล้ว scaffold อัตโนมัติ

💡
กฎ: ถ้าวิชาใหม่ใช้โครงเหมือนเดิม → ใช้ pattern เดิม · ถ้าปรับโครงจนต่าง → บันทึกเป็น pattern ใหม่ (fork) · วิชาถัดไปใช้ pattern ใหม่ได้เลย

Patterns ที่มีตอนนี้

PatternTypeใช้สำหรับ
astronomy-gamegameเกมสำรวจ · COSMOS LOG · EP01 narrative + EP02 game layer
wave-5e-poelesson-planแผน 5E+POE · Four-tier · Infographic ต่อแผน

โครงสร้างไฟล์ของ Pattern

.claude/patterns/<pattern-name>/
├── manifest.md          ← frontmatter: name, base, description, template_sources
├── questions.json       ← คำถามที่ถามผู้ใช้ตอน /pattern-use
└── [templates]          ← ชี้ไฟล์จริงใน lessons/ (ไม่ duplicate)

.claude/patterns/_templates/
├── STATE.md             ← template STATE สำหรับวิชาใหม่ (public)
└── STATE_private.md     ← template private notes

🌱 Flow เปิดวิชาใหม่

[1] /pattern-list [2] /pattern-use astronomy-game bio → ผมแสดง 14 คำถาม · คุณตอบ YAML ครั้งเดียว [3] ผม scaffold lessons/bio/ + STATE.md + Register subjects [4] /game-page-new bio ep01 1 "..." ... [5] ถ้าปรับโครงจน fork ได้ /pattern-save bio --name=bio-exploration --base=astronomy-game
ℹ️
ทำไม batch Q&A: ตอบทั้งหมดในข้อความเดียว ไม่ต้องถามทีละข้อ → ลด round-trip ~5x · ประหยัด token

🧠 State & Memory — Living memory ต่อวิชา

แต่ละวิชามี STATE.md ที่เก็บ "ตอนนี้ทำถึงไหน · story ล่าสุด · decisions" · ใช้ตอนเปิด chat ใหม่เพื่อให้ผมรู้บริบททันที ไม่ต้องอ่านไฟล์ทั้งหมด

🎯 ปัญหาที่ State แก้

ปัญหาวิธีแก้
เปิด chat ใหม่ · ไม่รู้ทำถึงไหน/catchup <subject> โหลด briefing
Story เปลี่ยนระหว่างทาง · bible เก่าSTATE.md เป็น live · override bible
ตัดสินใจไปแล้วลืมDecisions log append-only · มีประวัติ
MEMORY.md global บวมGlobal เก็บ pointer · content อยู่ที่ STATE.md

🏗️ 2 ชั้น (Public + Private)

🌐 Public — lessons/<subject>/STATE.md

ข้อมูลทางเทคนิค · commit ไป git · มีประวัติ · ย้ายเครื่องอัตโนมัติ

  • Story state, decisions, next tasks
  • Completed list
  • Open questions (ทั่วไป)

🔒 Private — _private/state/<subject>_notes.md

ไม่ commit · ส่วนตัว 100% · _private/ อยู่ใน .gitignore แล้ว

  • ความเห็นเกี่ยวกับนักเรียน
  • Idea ยังไม่ตกผลึก
  • Budget / ปัญหาส่วนตัว

💰 ประหยัด token อย่างไร

เทียบกับประหยัด
อ่าน bible.html ทั้งไฟล์STATE.md ~100 บรรทัด vs bible ~500 บรรทัด · ลด 80%
อ่านทุกหน้าเพื่อเข้าใจ storySTATE.md สรุปให้แล้ว · ลด 95%
เล่า context ทุก chat/catchup โหลดครั้งเดียว
⚠️
ข้อควรระวัง
• STATE.md ห้ามยาวเกิน 200 บรรทัด · ย้ายส่วน Done เก่าไป _history/
• ไม่ใส่รายละเอียดเนื้อหาแต่ละหน้า · แค่ index + decisions
• Decisions log = append-only · ห้ามลบของเก่า

💰 กลยุทธ์ประหยัด Token

กลยุทธ์วิธีผล
ระบุ path ตรงเป้าคำสั่งบอกไฟล์ที่ต้องอ่านลด 50-70%
Edit > Writeส่ง diff แทนไฟล์เต็มลด 80-95%
Line rangeRead offset=1285 limit=50ลด 90%+
Pattern = memoryโหลด lazy · update ครั้งเดียวลดซ้ำ
--no-confirmไม่ถามย้ำใน macroลด 20-30%
Batch Q&Aตอบชุดเดียว (pattern-use)ลด round-trip 5x
STATE.md briefing/catchup แทนอ่านทั้ง projectลด 90%
Crop screenshotเฉพาะจุดผิด~1k vs 2.5k
บอก element/id"แก้ canvas#em-wave"~50 vs 2000
